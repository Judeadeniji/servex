# Rule: Architecture Guidelines

## Context

ServeX's architecture revolves around AOT compilation of the route tree via `SonicRouter`. Any circumvention of this architecture defeats the framework's core value proposition.

## Agent Directives

### Router Integrity

- **Rule:** `SonicRouter` (Trie JIT) is the single source of truth for routing.
- **Agent Action:** You are strictly forbidden from adding ad-hoc matching logic, regex checks, or dynamic route parsing inside `fetch.ts`. All routing must be compiled into the single function string generated by `SonicRouter` at startup.

### Static File Handling

- **Rule:** Static routes bypass the router entirely.
- **Agent Action:** Inject static or pre-rendered routes directly into `Bun.serve({ static: app.static })`. Do not attempt to wrap static file serving in the router pipeline or middleware chain.

### Middleware Execution

- **Rule:** Strictly follow the `(ctx, next)` middleware pattern.
- **Agent Action:** In the AOT compiler, only allocate a `next` closure when `handlers[j].length > 1`. You must preserve this exact condition check. Do not allocate closures blindly.

### Internal Data Structures

- **Rule:** Use flat object literals internally.
- **Agent Action:** When designing internal (non-user-facing) data structures, use flat objects instead of nested objects or class instances to reduce pointer chasing and memory fragmentation. Do not add new abstractions (classes, factories, registries) to the routing layer without explicit benchmark justification from the user.
